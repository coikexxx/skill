# Skill Repository Checklist

Use this checklist after adding or editing a skill:

1. Confirm the skill lives in `skills/<skill-name>/`.
2. Confirm `SKILL.md` exists and matches the folder name.
3. Confirm optional resources stay inside the same skill folder.
4. Refresh shared indexes or README sections that enumerate skills.
5. Verify package status is stated accurately (`packaged` vs `source only`).
